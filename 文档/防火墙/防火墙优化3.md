# 三、整改方案

## 整改项一：HTTP Method防护

### 问题

允许危险HTTP方法。

### 风险方法

TRACE

TRACK

DEBUG

CONNECT

### 实施方案

建立允许列表：

```lua
local allowed_methods = {
    GET=true,
    POST=true,
    PUT=true,
    DELETE=true,
    OPTIONS=true,
    HEAD=true
}
```

检测：

```lua
local method = ngx.req.get_method()

if not allowed_methods[method] then
    ngx.exit(444)
end
```

### 预期效果

阻断：

- TRACE测试
- TRACK测试
- DEBUG测试

---

## 整改项二：Request Smuggling防护

### 问题

未检测：

```http
Content-Length
Transfer-Encoding
```

冲突。

### 实施方案

检测：

```lua
local headers = ngx.req.get_headers()

if headers["Transfer-Encoding"]
and headers["Content-Length"] then

    block()
end
```

### 扩展检测

检测：

```http
chunked
identity
```

混合编码。

### 预期效果

阻断：

- CL.TE
- TE.CL

攻击

---

## 整改项三：统一URL解码

### 问题

存在双重编码绕过。

例如：

```text
%2527
```

```text
u%6eion
```

### 实施方案

统一解码两次：

```lua
local uri = ngx.var.request_uri

uri = ngx.unescape_uri(uri)
uri = ngx.unescape_uri(uri)
```

然后执行规则匹配。

### 预期效果

解决：

- 双重编码
- 部分混淆绕过

---
