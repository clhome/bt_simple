使用说明

【御风进程守护】提示： 这里的服务与系统原生服务完全隔离，底层强制绑定 YuFeng 标签。删除或修改此处配置不会影响系统正常运行，请放心使用。

一般使用方法
  • 添加守护服务：点击上方【添加守护服务】按钮，根据需要填写配置。
  • 配置模式选择：
    • 极简向导模式 (推荐)：填入服务名称、运行用户、项目路径(绝对路径)和启动命令即可。
    • 高级代码模式：适合高级定制。直接编写 Systemd 配置文件，后台会自动附加 Documentation=tag:YuFeng 标签以实现安全接管。
  • 日常服务管理：在列表中可以一键启动、停止、重启服务，切换自启状态，以及查看运行日志。
Python 虚拟环境使用方法 (venv)

如果您的 Python 项目需要使用虚拟环境隔离依赖，请参考以下最佳实践进行配置：

  • 在终端中创建虚拟环境并安装依赖
    # 1. 进入您的项目目录
    cd /www/wwwroot/my_python_project

    # 2. 创建名为 venv 的虚拟环境
    python3 -m venv venv

    # 3. 激活虚拟环境并安装 requirements.txt 依赖
    source venv/bin/activate
    pip install -r requirements.txt
  • 在御风进程守护中进行配置：
    • 项目路径：填写您项目的绝对路径（如 /www/wwwroot/my_python_project）。
    • 启动命令：极其重要！请勿在启动命令中使用 source venv/bin/activate，亦不要使用相对路径。您必须直接调用虚拟环境内 Python 解释器的绝对路径。
    💡 启动命令配置示例
    标准 Python 脚本启动：
    /www/wwwroot/my_python_project/venv/bin/python main.py

    Django/Gunicorn 服务启动：
    /www/wwwroot/my_python_project/venv/bin/gunicorn -w 4 -b 127.0.0.1:8000 myproject.wsgi:application
